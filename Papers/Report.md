## Введение

**Актуальность темы.** В эпоху цифровизации инженерной деятельности значительные объемы технической документации, накопленные за десятилетия, переводятся в электронный вид путем сканирования. Однако этот процесс создает лишь графические копии, оставляя нерешенной задачу извлечения и структурирования ценных метаданных, содержащихся в основной надписи (штампе) чертежа. Ручной ввод этих данных — трудоемкий, медленный и подверженный ошибкам процесс. Применение алгоритмов машинного обучения для автоматизации распознавания и извлечения данных из ОН, является высокоактуальной задачей, направленной на повышение эффективности работы с электронными архивами и снижение влияния человеческого фактора.

**Объектом исследования** являются процессы и алгоритмы обработки изображений для извлечения текстовой информации из графических документов.

**Предметом исследования** является применение алгоритмов машинного обучения для разработки программного средства, реализующего конвейер для локализации и распознавания основной надписи на отсканированных технических чертежах.

**Целью** данной технологической практики является разработка программных алгоритмов с применением методов машинного обучения для автоматизации перевода аналоговой документации (чертежей) в цифровой вид.

Для достижения поставленной цели были определены следующие **задачи**:
1.  **Провести теоретические исследования**: изучить проблематику ручной обработки чертежей, проанализировать релевантные государственные стандарты (ГОСТ) и существующие подходы в области компьютерного зрения и машинного обучения.
2.  **Применить методы системного анализа**: определить границы проекта, выбрать технологический стек и спроектировать архитектуру программного конвейера.
3.  **Разработать программные модули**: реализовать алгоритмы предобработки изображений, локализации ОН и применить готовые ML-модели для извлечения текста.
4.  **Провести экспериментальные исследования**: протестировать работоспособность прототипа на наборе данных, оценить качество его работы.
5.  **Подготовить техническую документацию**, описывающую функциональность, структуру и принципы работы разработанного ПО.

**Методы исследования**: теоретический анализ научной литературы и стандартов, системный анализ и проектирование ПО, алгоритмизация, программирование с применением ML-библиотек, экспериментальное тестирование, анализ результатов.

### Раздел 1. Проектная часть

#### 1.1. Постановка производственной задачи

В ходе прохождения технологической практики в АО «Метрогипротранс» была поставлена задача оптимизации процессов обработки архивной технической документации. В ходе деятельности института возникает необходимость обработки больших массивов отсканированных чертежей. Текущий процесс ввода атрибутивной информации (шифр, номер листа и т.д.) выполняется сотрудниками вручную, что приводит к значительным временным затратам и риску возникновения ошибок из-за человеческого фактора.

Проект направлен на решение данной производственной задачи по автоматизации извлечения атрибутивной информации. Целью проекта является создание программного инструмента, который бы взял на себя рутинные операции по поиску основной надписи на чертеже и распознаванию ее содержимого, оставляя за сотрудником только функцию финальной проверки (валидации) результатов.

#### 1.2. Роль в проекте и организация работы

Работа выполнялась в рамках отдела информационных технологий (гипотетически) под руководством начальника отдела и при консультационной поддержке инженеров архивного отдела. В рамках проекта на меня были возложены функции инженера-разработчика, занимающегося исследованием и прототипированием решения.

**Выполняемые трудовые функции:**
*   **Анализ требований:** Взаимодействие с представителями архивного отдела для уточнения требований к извлекаемым данным.
*   **Исследование технологий:** Анализ существующих библиотек и фреймворков в области Computer Vision и Machine Learning (в частности, OpenCV и EasyOCR).
*   **Разработка прототипа:** Проектирование архитектуры ПО, написание кода, реализующего конвейер обработки.
*   **Тестирование:** Проверка работоспособности прототипа на предоставленных примерах чертежей.
*   **Документирование:** Подготовка технической документации и отчета по практике.

### Раздел 2. Практическая часть

#### 2.1. Аналитический обзор и постановка задачи

Проблема автоматизации ввода данных с бумажных носителей является классической для многих отраслей. В инженерной сфере она осложняется строгими требованиями к формату документации, регламентированными стандартами ГОСТ. Решение данной задачи лежит в области компьютерного зрения и применения алгоритмов машинного обучения.

Современные подходы к распознаванию текста на изображениях (OCR) основаны на глубоких нейронных сетях, таких как сверточные (CNN) и рекуррентные (RNN) сети. Библиотека EasyOCR, выбранная для данного проекта, использует именно такой подход (архитектура CRNN), что позволяет классифицировать ее как инструмент машинного обучения.

#### 2.2. Проектирование информационной системы

Архитектура разработанного решения представляет собой последовательный конвейер обработки данных (pipeline), включающий модуль предобработки, модуль локализации и модуль интеллектуального анализа.

##### 2.2.1. Математические основы и алгоритмы

**1. Коррекция перекоса (Deskew) с помощью преобразования Хафа.**
Для выравнивания изображения используется вероятностное преобразование Хафа. Метод преобразует точки на изображении из декартовой системы координат `(x, y)` в параметрическое пространство `(ρ, θ)`, где каждая точка соответствует прямой, заданной уравнением в нормальной форме:
`ρ = x * cos(θ) + y * sin(θ)`
где `ρ` — расстояние от начала координат до прямой, `θ` — угол перпендикуляра к прямой. Пики в пространстве Хафа (аккумуляторе) соответствуют доминирующим прямым на изображении. В данном проекте анализируются пики, соответствующие горизонтальным линиям, и вычисляется средний угол их наклона, на который затем поворачивается изображение.

**2. Фильтрация контуров для локализации ОН.**
После бинаризации изображения и поиска всех замкнутых контуров производится их фильтрация по геометрическим критериям:
- **Аппроксимация формы:** Контур аппроксимируется многоугольником с помощью алгоритма Рамера-Дугласа-Пекера. Для дальнейшего анализа отбираются только контуры, имеющие 4 вершины.
- **Фильтр по площади:** Отсеиваются слишком мелкие и слишком крупные контуры. `S_min < S_contour < S_max`, где `S` — площадь контура в пикселях.
- **Фильтр по соотношению сторон (Aspect Ratio):** Для оставшихся четырехугольников вычисляется соотношение ширины к высоте.
  `AR = width / height`
  Полученное значение `AR` сравнивается с эталонными значениями для штампов по ГОСТ (≈3.36 для формы 3 и ≈12.33 для формы 6) с некоторым допуском `ε`:
  `|AR - AR_gost| < ε`

**3. Архитектура нейросети для распознавания текста.**
Библиотека `EasyOCR` использует модель на основе архитектуры CRNN (Convolutional Recurrent Neural Network), которая хорошо зарекомендовала себя в задачах распознавания текста на изображениях. Архитектура состоит из трех основных частей:
- **Сверточные слои (CNN):** Выполняют роль извлекателя признаков (feature extractor). На вход подается изображение, и на выходе получается карта признаков.
- **Рекуррентные слои (RNN):** Карта признаков преобразуется в последовательность, которая обрабатывается рекуррентными слоями (обычно LSTM), способными улавливать контекстуальные зависимости в последовательности.
- **Слой транскрипции (CTC Loss):** Connectionist Temporal Classification — это функция потерь, используемая для обучения модели, которая позволяет работать с разметкой "текст на картинке" без необходимости точного позиционирования каждого символа.

### Раздел 3. Технико-экономическое обоснование

Целью данного раздела является оценка экономической целесообразности внедрения разработанного программного средства `gost-ocr`.

**1. Расчет трудозатрат на ручную обработку**
Исходные данные:
- Среднее время обработки одного чертежа вручную: 5 минут (0.083 часа).
- Средняя месячная зарплата сотрудника архива (условно): 50 000 руб.
- Количество рабочих часов в месяце: 168.
- Стоимость одного часа работы сотрудника: 50 000 / 168 ≈ 298 руб./час.

Стоимость ручной обработки одного листа:
`C_manual = 0.083 ч * 298 руб./ч ≈ 24.73 руб.`

**2. Расчет трудозатрат на автоматизированную обработку**
Исходные данные:
- Время машинной обработки одного чертежа: ~10 секунд.
- Время на верификацию результата оператором: ~20 секунд.
- Общее время: 30 секунд (0.0083 часа).

Стоимость автоматизированной обработки одного листа:
`C_auto = 0.0083 ч * 298 руб./ч ≈ 2.47 руб.`

**3. Оценка экономического эффекта**
Экономия на обработке одного листа:
`E_sheet = C_manual - C_auto = 24.73 - 2.47 = 22.26 руб.`

Предположим, в месяц обрабатывается архив из 2000 листов.
- Затраты при ручной обработке: `2000 * 24.73 = 49 460 руб.`
- Затраты при автоматизированной обработке: `2000 * 2.47 = 4 940 руб.`

**Годовой экономический эффект** от внедрения системы (при обработке 24 000 листов в год) составит:
`E_year = (49 460 - 4 940) * 12 = 44 520 * 12 = 534 240 руб.`

Данный расчет является предварительным, но наглядно демонстрирует высокую экономическую эффективность проекта, которая достигается за счет многократного сокращения трудозатрат на рутинные операции.

## Приложение А. Листинги программного кода

**А.1. Модуль локализации (`src/gost_ocr/localization.py`)**
```python
# Содержимое файла src/gost_ocr/localization.py
import cv2
import numpy as np

from gost_ocr.config import GOST_ASPECT_RATIO_TOLERANCE, GOST_STAMP_ASPECT_RATIOS


def find_stamp_contour(processed_image: np.ndarray) -> np.ndarray | None:
    """Находит контур штампа на предобработанном изображении."""
    contours, hierarchy = cv2.findContours(
        processed_image, cv2.RETR_TREE, cv2.CHAIN_APPROX_SIMPLE
    )

    if hierarchy is None:
        return None

    for i, contour in enumerate(contours):
        # Родительский контур без родителя
        if hierarchy[0][i][3] == -1:
            perimeter = cv2.arcLength(contour, True)
            approx = cv2.approxPolyDP(contour, 0.02 * perimeter, True)

            if len(approx) == 4:
                x, y, w, h = cv2.boundingRect(approx)
                if w > 100 and h > 20:  # Фильтр по минимальным размерам
                    aspect_ratio = w / float(h)
                    for _, ratio in GOST_STAMP_ASPECT_RATIOS.items():
                        if abs(aspect_ratio - ratio) < GOST_ASPECT_RATIO_TOLERANCE:
                            return approx
    return None
```

**А.2. Модуль предобработки (`src/gost_ocr/preprocessing.py`)**
```python
# Содержимое файла src/gost_ocr/preprocessing.py
import cv2
import numpy as np


def deskew(image: np.ndarray) -> np.ndarray:
    """Выравнивает изображение, устраняя перекос."""
    gray = cv2.cvtColor(image, cv2.COLOR_BGR2GRAY)
    gray = cv2.bitwise_not(gray)
    lines = cv2.HoughLinesP(
        gray, 1, np.pi / 180, 100, minLineLength=100, maxLineGap=10
    )
    if lines is None:
        return image

    angles = []
    for line in lines:
        x1, y1, x2, y2 = line[0]
        angle = np.rad2deg(np.arctan2(y2 - y1, x2 - x1))
        angles.append(angle)

    # Вычисляем медианный угол и поворачиваем
    median_angle = np.median(angles)
    (h, w) = image.shape[:2]
    center = (w // 2, h // 2)
    rotation_matrix = cv2.getRotationMatrix2D(center, median_angle, 1.0)
    rotated = cv2.warpAffine(
        image,
        rotation_matrix,
        (w, h),
        flags=cv2.INTER_CUBIC,
        borderMode=cv2.BORDER_REPLICATE,
    )
    return rotated
```
