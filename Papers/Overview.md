## Введение: Проблематика автоматизации обработки метаданных архивных чертежей

### 1. Контекст проблемы: цифровизация исторических архивов

Проектные институты, включая АО «Метрогипротранс», накопили огромные архивы проектной документации на бумажных носителях. В рамках программы цифровизации эти архивы сканируются, но это создает новую проблему: **сканирование преобразует документ в изображение, но не в структурированные данные**.

В результате:
- Сканы хранятся под техническими именами (`scan_001.jpg`).
- Поиск конкретного листа требует ручного просмотра тысяч файлов.
- Интеграция с системами электронного документооборота (СЭД) невозможна без ручного ввода метаданных.

Эта рутинная работа отнимает до 30% времени сотрудников архивных подразделений, отвлекая их от более важных задач.

### 2. Описание рутинной операции, подлежащей автоматизации

Типовой сценарий ручной обработки архивного скана:

1.  Архивариус получает папку с сотнями сканов.
2.  Для каждого файла:
    - Открывает изображение.
    - Визуально ищет основную надпись (штамп).
    - Считывает и перепечатывает шифр проекта и номер листа.
    - Переименовывает файл и вносит данные в реестр Excel.
3.  На 500 листов уходит 3–5 рабочих дней.

**Проблема**: задача монотонна, подвержена человеческим ошибкам и не масштабируется.

## GOST-OCR: Решение для автоматизации

**GOST-OCR** — это CLI-утилита на Python, которая автоматизирует этот процесс, предоставляя конвейер для обработки изображений.

### Возможности

*   **Реализует трехступенчатый конвейер:**
    1.  **Предобработка:** Корректирует поворот и перекос изображения, позволяет выбрать интересующую область (ROI) для поиска.
    2.  **Локализация штампа:** Используя методы компьютерного зрения (анализ контуров в OpenCV), находит на чертеже рамку штампа.
    3.  **Извлечение текста:** Распознает весь текст внутри найденного штампа с помощью движка `EasyOCR`.
*   **Пакетная обработка:** Работает с отдельными файлами (`.png`, `.jpg`, `.jpeg`) или с целой папкой изображений.
*   **Структурированный вывод:** Сохраняет все распознанные текстовые блоки, их координаты и уверенность распознавания в виде JSON-файла для каждого изображения.
*   **Режим отладки:** Позволяет сохранять промежуточные изображения на каждом этапе для визуального контроля.

### Ограничения

| Ограничение | Обоснование |
| :--- | :--- |
| **Только метаданные из штампа** | Геометрия чертежа (линии, размеры, схемы) сознательно не анализируется. Эта задача требует сертифицированного ПО и экспертизы инженера. |
| **Не является заменой человеку** | Инструмент предназначен для автоматизации рутины. Результаты, особенно с низкой уверенностью, требуют верификации человеком. |
| **Не для BIM-интеграции** | Извлеченные данные не предназначены для автоматической генерации 3D-моделей без ручной проверки. |

## Установка

Для работы проекта требуется Python 3.12+.

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/Sapar-hub/gost-ocr.git
    cd gost-ocr
    ```

2.  **Создайте и активируйте виртуальное окружение:**
    ```bash
    python -m venv .venv
    source .venv/bin/activate
    # Для Windows: .venv\Scripts\activate
    ```

3.  **Установите зависимости:**
    Проект использует `uv` для управления зависимостями.
    ```bash
    pip install uv
    uv pip sync pyproject.toml
    ```

## Использование

Утилита запускается из командной строки. Основная команда — `pipeline`.

### Полный цикл обработки

Эта команда выполняет все три этапа и сохраняет результаты в папку `output`.

```bash
gost-ocr pipeline /путь/к/вашим/изображениям/ --debug
```

**Аргументы и опции:**

*   `PATH` (обязательный): Путь к одному файлу или к папке с изображениями.
*   `--roi [POSITION]`: Указывает, какую часть изображения анализировать.
    *   **Доступные позиции:** `top`, `bottom`, `left`, `right`, `top_left`, `top_right`, `bottom_left`, `bottom_right` (по умолчанию).
*   `--flip` или `-f`: Пробовать все повороты изображения (0°, 90°, 180°, 270°).
*   `--debug` или `-d`: Включить режим отладки для сохранения промежуточных результатов.

## Структура проекта

*   `src/gost_ocr/cli.py`: Интерфейс командной строки.
*   `src/gost_ocr/preprocessing.py`: Модуль предобработки изображений.
*   `src/gost_ocr/localization.py`: Модуль поиска штампа.
*   `src/gost_ocr/extraction.py`: Модуль извлечения текста (`EasyOCR`).
*   `src/gost_ocr/config.py`: Файл с константами и путями.
*   `pyproject.toml`: Зависимости проекта.

## Формат вывода

*   **`output/`**: Здесь создаются `.json` файлы с результатами распознавания.
*   **`debug/`**: Здесь создаются папки с промежуточными изображениями для отладки.
